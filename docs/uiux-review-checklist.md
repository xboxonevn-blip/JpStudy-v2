# UI/UX Review Checklist

Use this checklist for each screen before merge.

## Discover

- [ ] Problem statement is clear (who, why, what outcome).
- [ ] Wireframe covers all key states (empty/loading/error/success).
- [ ] Information hierarchy is obvious without color.

## Visual

- [ ] Spacing and typography are consistent with current app style.
- [ ] Contrast is readable (text and controls).
- [ ] Touch targets are comfortable on mobile (>=44 px).
- [ ] Primary and secondary actions are clearly separated.

## Validate

- [ ] Tested on desktop and mobile layout sizes.
- [ ] Scroll and navigation behavior are smooth.
- [ ] Interactive states checked (pressed/disabled/focus).
- [ ] User feedback notes captured in `docs/uiux-progress.md`.

## Handoff

- [ ] Final acceptance criteria are met.
- [ ] Route and entry point are discoverable.
- [ ] Follow-up tasks logged in roadmap/backlog.
